## 3. 학습 진행 과정

### 3.1 1차 시도 — 기본 설정
- 기본 하이퍼파라미터로 100 에폭 학습
- **결과**: Val 80.79%, Train 99.8% → Train/Val 격차 19%p, 심각한 과적합

### 3.2 2차 시도 — 하이퍼파라미터 개선 (50 에폭)
- label_smoothing 0.1, weight_decay 1e-3, batch_size 64 적용
- **결과**: Val 82.55%, 격차 17%p로 소폭 개선. 여전히 과적합

### 3.3 3차 시도 — 통합 학습 + 정규화 종합 강화 (최종)
- `direction=None`으로 측면 데이터 포함 (4,800 → 9,600건)
- Augmentation 강화 (ColorJitter 상향, RandomGrayscale, GaussianBlur, RandomErasing 추가)
- dropout 0.5 → 0.55
- **결과**: Val **92.27%** (+9.72%p), 격차 epoch 14 기준 -1.8%p, 과적합 사실상 해소

## 5. 주요 이슈와 해결

### 이슈 1. 과적합 (Train/Val 격차 19%p)

**증상**: 1차 학습에서 Train 99.8%까지 상승했지만 Val은 80% 부근에서 정체. 모델이 학습 데이터를 외우고 검증 데이터에 일반화되지 않는 전형적 과적합 패턴.

**원인**:
- 정규화가 약함 (weight_decay 1e-4, dropout 0.5)
- batch_size 32로 작아 gradient 노이즈 큼
- 데이터 다양성 부족 (기본 augmentation만 적용)

**해결 단계**:
1. **정규화 강화** — weight_decay 1e-4 → 1e-3 (10배), dropout 0.5 → 0.55, label_smoothing 0.1 추가로 모델 과신 억제
2. **Augmentation 강화** — ColorJitter 강도 상향 + RandomGrayscale/GaussianBlur/RandomErasing 4종 추가로 입력 다양성 증가
3. **데이터 자체 2배 확장** — 측면 데이터 통합 (자세한 내용은 이슈 3 참고)
4. **배치 크기 상향** — 32 → 64로 학습 안정화

**결과**: 격차 19%p → 1.8%p (epoch 14). 최종 epoch 100 기준 9.2%p로 양호 범위 유지.

### 이슈 2. LR 스케줄 오염 (Resume 시)

**증상**: Colab 세션 만료 후 `--resume` 옵션으로 체크포인트에서 이어 학습했을 때, 학습이 비정상적으로 진행되며 성능이 불안정하게 등락.

**원인**: `CosineAnnealingLR` 스케줄러는 optimizer state와 별개로 자체 step 카운터를 가지는데, resume 시 체크포인트에 저장된 lr값이 무시되고 스케줄러가 epoch 0부터 다시 시작. 결과적으로 이미 감쇠된 lr이 다시 초기값(0.0005)으로 튀어 올라 학습 경로를 망가뜨림.

**해결**: Resume 전략 포기. 처음부터 끝까지 한 번에 학습 완주하는 방식으로 변경. `early_stopping_patience`도 무한으로 설정해 스케줄이 의도대로 100 에폭에 걸쳐 0까지 감쇠하도록 보장.

**결과**: LR이 epoch 1부터 100까지 cosine 곡선대로 정상 감쇠. 학습 후반부 fine-tuning 효과가 제대로 발휘됨.

### 이슈 3. 분리 학습 vs 통합 학습 전략 선택

**배경**: AI Hub 08-14 데이터는 정면(front)과 측면(side) 두 방향으로 구성되어 있어, "정면/측면 각각 전용 모델을 학습할지" vs "하나의 모델로 통합 학습할지" 선택 필요.

**분리 학습의 장단점**
- 장점: 각 모델이 해당 각도에 특화 → 개별 정확도 소폭 높을 가능성
- 단점: 모델 2개 유지로 서버 메모리/추론 비용 2배, 입력 이미지 각도 판별 로직 필요, 체크포인트/평가 파이프라인 2배 복잡화

**통합 학습의 장단점**
- 장점: 단일 모델로 단순한 파이프라인, 정면/측면 구분 없이 즉시 추론 가능, 두 각도의 공통 특징을 함께 학습해 실제 이미지 다양성에 강건
- 단점: 정면 전용 모델보다 정면 정확도가 소폭 낮을 가능성

**선택: 통합 학습**

피부질환은 어느 각도에서 촬영하든 병변의 특징(색상, 질감, 형태)이 동일하므로 분리할 임상적 근거가 약함. 의료 보조 서비스 특성상 사용자가 어떤 각도로 찍을지 예측 불가하므로 **단일 모델이 실용적**.

**구현**: 데이터셋 로더 `AihubFacialDataset`의 `direction` 파라미터 기본값이 `"front"`라 측면 데이터가 자동 필터링되고 있던 문제를 발견, 학습·평가 스크립트 모두 `direction=None` 명시로 전환.

**결과**: 학습 데이터 4,800 → 9,600건 (2배), Val 82.55% → 92.27% (+9.72%p). **세 가지 이슈 중 가장 큰 성능 개선 요인**.

## 7. 결론

- AI Hub 가이드라인 목표(80%)를 **12%p 초과 달성** (92.17%)
- 과적합 문제를 **정규화 강화 + Augmentation 강화 + 통합 학습** 3가지 조합으로 해결
- **측면 데이터 통합(통합 학습)이 가장 큰 개선 요인** (+9.72%p)
- Threshold 최적화로 Macro F1 0.9222 → 0.9437 추가 향상 (불확실 판정 3.25%만으로)
