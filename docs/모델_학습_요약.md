# SCIN 피부 질환 분류 모델 학습 요약

## 프로젝트 개요

**목표**: SCIN 데이터셋을 활용한 다중 라벨 피부 질환 이미지 분류 모델 개발
**모델 아키텍처**: ResNet50 (ImageNet Pretrained)
**데이터셋**: SCIN (Skin Condition Image Network)
**학습 프레임워크**: PyTorch 2.0+
**학습 환경**: Apple Silicon (MPS) / CUDA / CPU

---

| 구분 | 샘플 수 | 비율 |
|------|---------|------|

- **선택 기준**: 상위 50개 빈도 질환 (최소 샘플 10개 이상)

## 데이터 전처리

### 이미지 전처리
```python
# 학습 시 (Train)
- Resize: 256x256
- RandomCrop: 224x224
- RandomHorizontalFlip: p=0.5
- ColorJitter: brightness=0.2, contrast=0.2, saturation=0.2, hue=0.1
- RandomRotation: degrees=15
- Normalize: mean=[0.485, 0.456, 0.406], std=[0.229, 0.224, 0.225] (ImageNet)

# 검증/테스트 시 (Val/Test)
- Resize: 224x224
- Normalize: mean=[0.485, 0.456, 0.406], std=[0.229, 0.224, 0.225]
```

### 라벨 인코딩
- **형식**: 다중 라벨 이진 행렬 (Multi-label Binary Matrix)
- **차원**: (N, 50) - N: 샘플 수, 50: 클래스 수
- **인코딩**: 각 샘플은 1개 이상의 질환 라벨을 가질 수 있음
- **평균 라벨 수**: 케이스당 약 1.2개

### 클래스 불균형 처리
- **가중치 계산 방식**: 역빈도 기반 (Inverse Frequency)
- **공식**: `weight = total_samples / (num_classes * class_count)`
- **가중치 범위**: 0.5 ~ 10.0 (소수 클래스에 높은 가중치)

## 학습 시도 및 결과

### 1차 학습 (Baseline)

**하이퍼파라미터**:
| 파라미터 | 값 |
|----------|-----|
| **손실 함수** | BCEWithLogitsLoss + 클래스 가중치 |
| **Optimizer** | Adam |
| **Learning Rate** | 0.0001 |
| **Batch Size** | 16 |
| **Epochs** | 100 |
| **Weight Decay** | 1e-3 |
| **Dropout** | 0.6 |
| **Patience** | 15 (Early Stopping) |
| **Scheduler** | ReduceLROnPlateau (factor=0.5, patience=5) |

**성능 결과** (Test Set):
| 메트릭 | 값 | 설명 |
|--------|-----|------|
| **F1-Score** | **0.0500** | Macro-averaged F1 |
| **Precision** | 0.0972 | 예측 정확도 매우 낮음 |
| **Recall** | 0.0368 | 실제 질환 감지율 낮음 |
| **Top-1 Accuracy** | 22.15% | 첫 번째 예측 정확도 |
| **Top-3 Accuracy** | 50.23% | 상위 3개 중 정답 포함 |
| **Top-5 Accuracy** | **64.84%** | 상위 5개 중 정답 포함 |

**분석**:
- **매우 낮은 F1-Score**: 5.0% (Precision-Recall 모두 낮음)
- **Precision 낮음**: 9.72% (과다 예측 또는 잘못된 예측)
- **Recall 매우 낮음**: 3.68% (실제 질환을 대부분 놓침)
- **Top-5 Accuracy**: 64.84%로 상위 5개 예측 중에는 정답이 포함되는 경우가 많음
- **문제점**: Threshold 0.5가 너무 높아 대부분의 예측을 걸러냄

**학습 소요 시간**: 약 1.5시간 (Apple Silicon MPS)

---

### 2차 학습 (Focal Loss)

**동기**: 1차 학습의 낮은 Recall 문제를 Focal Loss로 해결 시도

**하이퍼파라미터 변경**:
| 파라미터 | 1차 (Baseline) | 2차 (Focal Loss) | 변경 사유 |
|----------|---------------|------------------|-----------|
| **손실 함수** | BCEWithLogitsLoss | **Focal Loss** | 클래스 불균형 해결 |
| **Focal Alpha** | - | **0.25** | Minor class 가중치 |
| **Focal Gamma** | - | **2.0** | 쉬운 샘플 무시 |
| **Learning Rate** | 0.0001 | **0.0001** | 동일 |
| **Batch Size** | 16 | **16** | 동일 |
| **Dropout** | 0.6 | **0.6** | 동일 |

**성능 결과** (Test Set):
| 메트릭 | 1차 (Baseline) | 2차 (Focal Loss) | 변화 |
|--------|---------------|------------------|------|
| **F1-Score** | 0.0500 | **0.0852** | **+70%** |
| **Precision** | 0.0972 | 0.0489 | -50% |
| **Recall** | 0.0368 | 0.4861 | **+1221%** |
| **Top-1 Accuracy** | 22.15% | 34.47% | **+55%** |
| **Top-3 Accuracy** | 50.23% | 61.42% | **+22%** |
| **Top-5 Accuracy** | 64.84% | 70.09% | **+8%** |

**분석**:
- **Recall 대폭 향상**: 3.68% → 48.61% (+1221%) - Focal Loss가 질환 감지율 크게 개선
- **F1-Score 개선**: 0.0500 → 0.0852 (+70%)
- **Top-K Accuracy 향상**: Top-1 +55%, Top-3 +22%, Top-5 +8%
- **Precision 저하**: 9.72% → 4.89% (-50%) - 과다 예측 경향 증가
- **Trade-off**: Recall 향상 대신 Precision 희생

**주요 문제점**:
- **Major Class 과적합**: Eczema, Allergic Contact Dermatitis 등 다수 클래스에 과적합
  - Eczema Recall: **100%** (모든 샘플을 Eczema로 예측)
  - Eczema Precision: **32%** (과다 예측)
- **Alpha/Gamma 파라미터 과도**: alpha=0.25, gamma=2.0이 너무 강해 Major class 편향

**결론**: Recall은 개선되었으나 Precision이 너무 낮아 실용성 부족 → 파라미터 조정 필요 → 아카이브 처리 (`archive/focal_loss_failed/`)

---

### 3차 학습 (Balanced Focal Loss)

**동기**: 2차 학습의 Precision 저하 문제 해결 → Alpha/Gamma 파라미터 완화

**하이퍼파라미터 변경**:
| 파라미터 | 2차 (Focal Loss) | 3차 (Balanced) | 변경 사유 |
|----------|-----------------|----------------|-----------|
| **Focal Alpha** | 0.25 | **0.5** ↑ | Minor class 보호 강화 |
| **Focal Gamma** | 2.0 | **1.5** ↓ | Major class 과적합 완화 |
| **Learning Rate** | 0.0001 | **0.00005** ↓ | 더 안정적 학습 |
| **Batch Size** | 16 | **32** ↑ | 일반화 향상 |
| **Dropout** | 0.6 | **0.3** ↓ | 과적합 완화 |
| **Weight Decay** | 1e-3 | **1e-4** ↓ | L2 정규화 완화 |
| **Epochs** | 50 | **60** | 충분한 수렴 |

**성능 결과** (Test Set):
| 메트릭 | 1차 (Baseline) | 2차 (Focal Loss) | 3차 (Balanced) | 변화 (1차→3차) |
|--------|---------------|------------------|----------------|---------------|
| **F1-Score** | 0.0500 | 0.0852 | **0.1715** | **+243%** |
| **Precision** | 0.0972 | 0.0489 | **0.1052** | **+8%** |
| **Recall** | 0.0368 | 0.4861 | **0.5390** | **+1365%** |
| **Top-1 Accuracy** | 22.15% | 34.47% | **35.39%** | **+60%** |
| **Top-3 Accuracy** | 50.23% | 61.42% | **62.79%** | **+25%** |
| **Top-5 Accuracy** | 64.84% | 70.09% | **72.37%** | **+12%** |

**분석**:
- **최고 성능 달성**: F1-Score 0.1715 (1차 대비 +243%, 2차 대비 +101%)
- **Precision-Recall 균형**: Precision 10.52%, Recall 53.90%로 2차 대비 균형 개선
- **과적합 해소**: 2차 학습의 Major class 과적합 문제 해결
- **Recall 유지**: 2차의 높은 Recall(48.61%) 대부분 유지하면서 Precision 회복
- **Top-K Accuracy 최고**: Top-5 72.37% (실용적 성능)

**학습 소요 시간**: 약 2시간 (Apple Silicon MPS)

## 모델 평가 상세

### 평가 메트릭 설명

**F1-Score (Macro-averaged)**:
- Precision과 Recall의 조화평균
- 클래스 불균형을 고려한 전체 성능 지표
- **현재**: 0.1715 (17.15%)

**Top-K Accuracy**:
- 상위 K개 예측 중 정답이 포함되는 비율
- 다중 라벨 분류에서 중요한 지표
- **Top-5**: 72.37% (실용적 성능)

**Precision vs Recall**:
- **Precision 낮음 (10.52%)**: 과다 예측 → 거짓 양성 높음
- **Recall 높음 (53.90%)**: 실제 질환 감지율 높음 → 거짓 음성 낮음
- **Trade-off**: 의료 진단에서는 Recall이 더 중요 (질환 놓치지 않기)

### Threshold 최적화

**기본 Threshold**: 0.5 (Binary Classification)
- **1차 학습 문제**: Threshold 0.5가 너무 높아 대부분 예측 걸러짐 → Recall 3.68%, F1-Score 0.05
- 다중 라벨 분류에서 0.5는 부적합

**최적 Threshold**: 0.15 (Focal Loss 기준)
- **3차 학습 적용**: Threshold 0.15 사용 → Recall 53.90%, F1-Score 0.1715
- Recall 대폭 향상 (3.68% → 53.90%, +1365%)
- Precision 소폭 향상 (9.72% → 10.52%, +8%)
- **Trade-off**: 민감도 우선 (의료 진단에서 중요)

**Threshold 영향**:
| Threshold | F1-Score | Precision | Recall | 설명 |
|-----------|----------|-----------|--------|------|
| 0.5 | 0.05 | 9.72% | 3.68% | 너무 보수적 (1차 학습) |
| 0.15 | **0.1715** | **10.52%** | **53.90%** | 균형 (3차 학습) |
